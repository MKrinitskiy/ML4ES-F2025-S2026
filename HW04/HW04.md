# ML4ES 2025–2026

## Домашнее задание №4: оценка качества моделей классификации

---

## 1. Коллекция данных и постановка задачи

В данном задании используется синтетическая коллекция данных для задачи бинарной классификации.

Данные доступны для скачивания по [ссылке](https://ml4es.ru/links/ML4ES2025-HW04-dataset)

### Описание данных

- Каждый объект описывается набором числовых признаков;
- Целевая переменная — бинарная:
  - Класс 0  
  - Класс 1  

**ВАЖНО:**  
Данные обладают **существенным дисбалансом классов** (распределение классов неравномерно).

---

## 2. Формализация задания

В решении должны быть явно указаны:

- постановка задачи бинарной классификации;
- размерность и интерпретация признакового пространства;
- описание целевой переменной;
- способ разделения на train/test (если используется дополнительно);
- используемые метрики качества.

---

## 3. Цель задания

Цель задания — продемонстрировать понимание того, что:

- различные метрики качества могут давать **противоречивые оценки** одной и той же модели;
- метрика **Accuracy может быть неадекватной** в задачах с несбалансированными данными;
- метрика **F1-score лучше соответствует интуитивному представлению о качестве** модели;
- метрика **ROC AUC оценивает не конкретный обученный алгоритм классификации, а способность модели ранжировать объекты**.

---

## 4. Рассматриваемые модели

Необходимо реализовать и сравнить следующие модели:

### 4.1 Тривиальные модели (4 шт., обязательно)

1. Константные модели:
   - всегда назначает класс 0
   - всегда назначает класс 1
   
2. Случайные модели:
   - случайное назначение класса с вероятностью 0,5
   - случайное назначение класса с вероятностью, оцененной по данным

### 4.2 Обучаемые модели

3. Наивный байесовский классификатор с предположением о нормальности распределения признаков
3. Логистическая регрессия

Допускается использование `sklearn`
---

## 5. Оценка качества моделей

Для каждой модели необходимо вычислить:

- Accuracy  
- Precision  
- Recall  
- F1-score  

---

### 5.1 Анализ метрик

Для всех моделей необходимо:

- сравнить значения Accuracy;
- сравнить значения F1-score;
- сделать вывод о том, какая модель является лучшей по каждой метрике.

**Обязательный вопрос:**

> Совпадает ли «лучшая» модель по Accuracy с вашим интуитивным представлением о хорошей модели?

## 7. Вероятностная модель и порог

Для обучаемых вероятностных моделей:

- получить вероятности принадлежности к классу 1;
- рассмотреть различные значения порога вероятности для принятия решения о назначении метки класса;
- вычислить и продемонстрировать значения метрик качества при различных пороговых значениях вероятности.

### 7.1 Исследование порога

Необходимо:

- варьировать порог от 0 до 1 с малым шагом;
- для каждого значения порога вычислить:
  - Accuracy
  - Precision
  - Recall
  - F1-score

Построить график:

- F1-score от значения порога

**Обязательные вопросы:**

- При каком пороге достигается максимум F1?
- Совпадает ли этот порог с 0.5?
- Почему?

## 8. ROC-кривая и AUC

Для вероятностных моделей:

- построить ROC-кривую;
- вычислить AUC ROC.

### 8.1 Анализ ROC

Необходимо:

- интерпретировать форму ROC-кривой;
- сравнить с:
  - случайной моделью;
  - лучшим возможным случаем.

**Обязательные вопросы:**

- Что означает AUC ROC?
- Зависит ли AUC от порога классификации?
- Почему AUC можно интерпретировать как качество ранжирования объектов?
